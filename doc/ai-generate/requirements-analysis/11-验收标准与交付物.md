# 验收标准与交付物

## 模块定位

本章用于明确“做成什么算达标”。如果没有清晰验收标准，需求就无法转化为可执行开发目标和可验证交付结果。

## 一、功能验收标准

### 1. 智课生成模块

至少应完成：

- 课件上传与解析。
- 知识点结构预览。
- 结构化讲稿生成。
- 教师可编辑。
- 语音合成能力接入。

### 2. 问答交互模块

至少应完成：

- 文字问答。
- 语音问答链路中的语音转文字。
- 多轮问答支持。
- 高相关回答输出。

### 3. 进度续接模块

至少应完成：

- 进度记录。
- 理解程度判断。
- 问答后继续讲或补讲。
- 进度可视化或可展示结果。

## 二、指标验收标准

项目应满足或对齐以下口径：

- 课件解析响应时间 <= 2 分钟 / 份。
- 问答响应时间 <= 5 秒。
- 知识点识别准确率 >= 80%。
- 答案准确率 >= 85%。
- 样本集不少于 100 份，覆盖文、理、工三类课程。
- 功能接口不少于 10 个。

## 三、集成验收标准

- 能说明如何嵌入泛雅 Web 端或学习通移动端。
- 能说明课程、用户与智课对象的同步关系。
- 能说明异步解析和接口调用流程。
- 能展示统一接口风格和安全校验思路。

## 四、交付物清单

根据赛题说明，当前轮应准备的交付物至少包括：

- 项目概要介绍。
- 项目简介 PPT。
- 项目详细方案。
- 项目演示视频。
- 与平台集成方案说明。
- 原型图或集成原型说明。
- 关键模块设计与创新点文档。
- 可运行 Demo。

## 五、证据类交付要求

仅有功能展示还不够，还应提供可量化证据，例如：

- 解析耗时统计。
- 问答耗时统计。
- 样本构成表。
- 知识点识别与答案准确率评测结果。
- 接口清单与调用链路说明。
- 典型案例截图或录屏片段。

## 六、验收优先级结论

从验收角度看，最重要的不是某个单点功能是否做得花哨，而是以下四件事是否同时成立：

1. 主链路闭环能跑通。
2. 关键指标有证据。
3. 平台集成有标准接口口径。
4. 交付材料足够支撑演示与答辩。
