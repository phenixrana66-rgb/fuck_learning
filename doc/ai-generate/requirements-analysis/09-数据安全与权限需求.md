# 数据安全与权限需求

## 模块定位

数据安全与权限需求用于约束系统在平台接入、课件处理、问答交互和学习记录过程中，如何保护数据、限制访问和支持审计。

## 一、数据分类需求

当前轮至少涉及以下数据类别：

- 课件内容与解析产物。
- 课程信息与用户信息。
- 问答内容与语音转写结果。
- 学习进度与行为记录。
- 接口签名参数、身份令牌和请求日志。

不同数据虽然都服务于主链路，但敏感程度和保存要求并不相同，需要区别对待。

## 二、传输安全需求

- 对外接口推荐使用 HTTPS。
- 敏感字段在传输过程中应具备额外保护能力。
- 请求需带签名校验信息，避免非法调用和参数篡改。
- 请求需具备超时控制，防止恶意占用资源。

## 三、签名校验需求

根据开放 API 规范，接口需支持：

- `enc` 签名字段。
- 按参数名 ASCII 升序拼接非空参数。
- 结合固定密钥与时间参数计算校验结果。

需求层的重点不是规定实现代码，而是明确签名校验属于正式能力，而非可省略项。

## 四、存储安全需求

- 课件内容、学习数据和交互记录应按敏感级别做保护。
- 用户个人信息应遵循最小必要原则，不存储无关字段。
- 超出保留期限的数据应具备脱敏或删除策略。

## 五、权限控制需求

### 1. 角色模型

当前轮至少应区分：

- 学生
- 教师
- 平台管理员或平台调用方

### 2. 权限边界

- 教师应拥有课件上传、脚本编辑、发布确认等权限。
- 学生应拥有学习、提问、进度交互等权限。
- 平台方应拥有对象同步和受控调用权限。

### 3. 非法访问处理

- 未授权请求应返回明确错误。
- 签名校验失败和身份校验失败应可区分。

## 六、日志审计需求

- 关键接口调用应记录调用方、时间、操作内容和请求标识。
- 日志应服务于问题排查、安全审计和答辩举证。
- 需要支持通过 `requestId` 追踪问题。

## 七、隐私边界需求

教育场景下的问答内容、学习行为和联系方式等信息具有隐私属性，因此：

- 不应把学生问答与行为数据作为无边界收集对象。
- 不应为了演示方便暴露真实个人信息。
- 在提交案例、视频和样本时应具备脱敏意识。

## 八、需求结论

数据安全与权限需求不是平台接入之后才考虑的附属条款，而是会直接影响接口设计、数据范围、日志策略和最终可落地性的硬约束。
