# 范围边界与非目标

## 当前轮明确纳入范围的内容

基于赛题说明和开放 API 规范，当前轮需求基线明确纳入以下内容。

### 一、智课生成主链路

- 支持 PPT / PDF 课件接入。
- 能对课件进行结构化解析。
- 能提取知识点、重点、章节或节点关系。
- 能生成结构化讲稿。
- 支持教师编辑讲稿。
- 支持语音合成并形成可播放资源。

### 二、实时问答主链路

- 支持文字提问。
- 支持语音提问和语音转文字。
- 支持多轮问答。
- 回答需尽量与当前课件上下文高相关。
- 回答应能关联知识点、章节或页码证据。

### 三、进度续接与节奏调整主链路

- 记录学习进度。
- 结合问答结果识别理解程度。
- 支持继续讲、补讲和节奏调整。
- 支持在问答后定位原讲解节点并恢复学习。

### 四、平台对接与安全要求

- 支持课程同步与用户同步。
- 对齐 `/api/v1` 版本化接口规范。
- 使用统一响应结构。
- 支持签名校验、权限控制和日志审计。

### 五、验收与交付要求

- 样本集不少于 100 份。
- 接口不少于 10 个。
- 对齐解析时长、问答时长、准确率等指标。
- 形成项目方案、PPT、演示视频、原型与 Demo 等交付物。

## 当前轮明确不作为主线的内容

以下内容不是禁止讨论，而是明确不应成为当前轮的必做主线。

### 1. 数字人相关能力

- `数字人` 只能作为可扩展表现层预留。
- 当前轮不要求必须交付数字人讲授。
- 当前轮的核心仍是结构化讲授、问答和续接闭环。

### 2. 重型多智能体体系

- 不把系统设计目标扩展为复杂多 Agent 协作平台。
- 不要求构建炫技式智能体编排体系。

### 3. 自建模型或重型基础设施

- 不要求自建 LLM、TTS、ASR 服务。
- 不要求微服务化、Kubernetes、复杂消息系统等重型基础设施。

### 4. 与赛题主线无直接关系的增强功能

- 不增加脱离教学闭环的社交化、社区化或泛内容推荐功能。
- 不把平台运营分析、复杂 BI 能力作为当前轮强制需求。

## 边界判断原则

当新增想法出现时，统一按以下标准判断是否纳入当前轮：

1. 是否直接支撑“解析、生成、问答、续接”闭环。
2. 是否直接影响赛题硬指标、平台集成或验收交付。
3. 如果暂时不做，是否会阻断主链路。

只有同时满足主链路必要性或硬约束必要性的内容，才纳入当前轮必做范围。

## 前提假设与依赖条件

- 平台可提供基本课程和用户同步条件。
- 课件样本可合法获取并满足训练或评测需要。
- 项目可调用外部 AI 能力接口完成 LLM、TTS、ASR 相关能力。
- 当前轮重点是形成可演示、可验收的稳定方案，而不是追求终极产品化。
