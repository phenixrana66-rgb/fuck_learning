# 优先级风险与实施重点

## 一、优先级划分原则

当前轮的优先级判断，不应按“技术上想做什么”排序，而应按“是否影响闭环、指标和验收”排序。

## 二、P0 级需求，必须优先完成

### 1. 课件解析与结构化结果

原因：没有结构化结果，后续讲稿、问答、续接都会失去共同上下文。

### 2. 结构化讲稿生成与教师可编辑

原因：赛题要求智课生成，且教师侧必须具备可控性。

### 3. 基于当前章节的实时问答

原因：这是教学互动链路的核心，也是最容易被评审直接观察到的能力。

### 4. 问答后继续讲或补讲

原因：这是项目与普通问答系统拉开差距的关键点。

### 5. 平台对接的标准接口框架

原因：没有标准接口，系统很难被说明为可集成方案。

## 三、P1 级需求，应在主链路稳定后完成

- 语音问答完整体验优化。
- 更细的进度可视化。
- 指标统计自动化。
- PDF 兼容性增强。

这些内容重要，但应建立在主链路已稳定的前提下推进。

## 四、P2 级需求，可作为预留扩展

- `数字人` 表现层。
- 更复杂的多模态增强展示。
- 更重型的平台运营分析能力。

这些内容可以成为加分项或后续方向，但不能挤占当前轮主线资源。

## 五、主要风险

### 风险 1，做成“PPT 播放器 + 聊天机器人”

这会导致系统看似功能不少，但缺乏课程结构和续接机制，直接偏离赛题重点。

### 风险 2，回答看起来流畅，但无法回到课件证据

一旦不能说明答案来源，可信度和答辩说服力都会显著下降。

### 风险 3，过早投入扩展能力

如果把主要精力投入数字人或复杂表现层，主链路反而可能跑不通。

### 风险 4，接口和平台要求后置处理

如果平台集成与安全规范拖到最后，会导致返工风险很大。

### 风险 5，只有功能，没有证据

即使系统能演示，如果缺少时间、准确率、样本和接口文档等证据，也很难在验收中形成完整说服力。

## 六、实施重点建议

### 第一优先，先把结构跑通

把 PPT / PDF 转成稳定的结构化预览，是整个项目的基础。

### 第二优先，跑通最小智课闭环

先做到“解析 -> 讲稿 -> 编辑 -> 发布”，再逐步优化表现。

### 第三优先，跑通证据问答

优先保证回答相关且有来源，再考虑更丰富表达。

### 第四优先，补足续接与指标

续接和评测证据是拉开差距与完成验收的关键。

## 七、结论

当前轮的实施重点应始终围绕一句话展开：先把教学闭环跑通，再在闭环之上补性能、准确率和扩展能力。
