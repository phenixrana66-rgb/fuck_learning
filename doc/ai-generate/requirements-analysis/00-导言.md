# 需求分析文档集导言

本目录用于把 `doc/ai-generate/origin/塞题说明.md` 与 `doc/ai-generate/origin/超星AI互动智课服务系统开放API设计规范与示例.pdf` 重新整理为一套可直接指导后续方案设计、开发实施、测试验收和答辩准备的需求基线文档。

这套文档的定位，不是再写一份泛泛的项目介绍，也不是提前进入架构设计或技术选型，而是先回答四个根问题：

1. 这个系统到底要解决什么教学问题。
2. 教师、学生、平台方分别需要什么。
3. 当前轮必须做到什么程度，哪些能力只是预留扩展。
4. 后续如何判断系统已经达到赛题要求与交付要求。

## 来源与整理原则

本目录只以两类来源为准：

- 赛题说明给出的业务目标、三大核心模块、性能指标、样本规模和交付要求。
- 开放 API 规范给出的接口分组、版本约束、统一响应、签名校验、安全要求和平台对接方式。

整理过程中遵循以下原则：

- 先需求，后架构，再技术栈。
- 先闭环，后扩展。
- 先硬指标，后增强项。
- 先对齐赛题主线，避免写成脱离场景的通用 AI 产品。

## 这套文档回答什么，不回答什么

本目录重点回答：

- 教学闭环是什么。
- 三大核心模块分别需要哪些能力。
- 与平台和开放 API 如何对齐。
- 数据、安全、性能、验收口径如何定义。

本目录不展开以下内容：

- 具体分层架构、服务边界和数据表设计。
- 具体技术栈、第三方库和部署工具选型。
- 与当前轮目标无直接关系的炫技扩展。

这些内容会交给后续的 `project-architecture/` 与 `tech-stack-selection/` 目录处理。
