# 接口到代码文件映射

本文件用于回答一个非常具体的问题：

**看到某个 API 文档后，代码应该去哪里改。**

平时请按这个顺序使用：

1. 先读 [01-编码入口.md](./01-编码入口.md)
2. 再读 [02-后端模块与实现映射.md](./02-后端模块与实现映射.md)
3. 真正开始补某个接口时，再用本文件定位代码
4. 最后回到 `../api/` 看该接口的契约细节

## 一、通用定位规则

当前后端默认按 domain 分目录，每个接口一般至少关联三类文件：

- `router.py`：路由入口、参数接收、统一响应
- `schemas.py`：请求/响应 DTO
- `service.py`：业务逻辑与跨模块编排

如果本次改动不只是示例逻辑，而是开始接真实依赖，默认继续补这些文件：

- `repository.py`：数据库访问
- `models.py`：ORM 模型
- `tasks.py`：长耗时后台任务
- `providers.py`：外部能力调用（LLM / TTS / ASR / 对象存储 / 外部平台）

## 三、课件解析接口

| API 文档 | 当前路由文件 | 当前 Schema 文件 | 当前 Service 文件 | 相关协作模块 | 后续常见新增文件 |
|---|---|---|---|---|---|
| `api/lesson-parse-post.md` | `backend/app/courseware/router.py` | `backend/app/courseware/schemas.py` | `backend/app/courseware/service.py` | `backend/app/parser/service.py` `backend/app/cir/service.py` `backend/app/tasks/service.py` | `backend/app/courseware/repository.py` `backend/app/courseware/models.py` `backend/app/courseware/tasks.py` 或 `backend/app/tasks/` `backend/app/parser/providers.py` |
| `api/lesson-parse-get-by-parseId.md` | `backend/app/courseware/router.py` | `backend/app/courseware/schemas.py` | `backend/app/courseware/service.py` | 当前直接读取 `backend/app/courseware/service.py` 内存结果 | `backend/app/courseware/repository.py` `backend/app/tasks/repository.py` |

这组接口的核心规律是：

- `courseware/` 是对外编排入口；
- `parser/` 提供解析能力；
- `cir/` 提供课程中间表示；
- `tasks/` 提供任务状态记录。

当前已落地的最小真实解析路径：

- `backend/app/parser/pptx_reader.py`：读取 `.pptx` 并提取 slide 文本
- `backend/app/parser/llm_client.py`：调用 OpenAI 兼容接口生成章节结构
- `backend/app/parser/service.py`：把 LLM 结果转成 `StructurePreview`
- `backend/app/cir/service.py`：把 preview 转成 `CIR`

当前要特别注意：

- 接口字段仍是 `fileType=ppt`，但 demo 实际只接受 `.pptx`
- 当前解析在 `POST` 内同步完成，所以 `taskStatus` 已直接为 `completed`
- 若继续把它改成真实异步，不只要改 `tasks/`，还要回看 `api/lesson-parse-*.md`

这组接口最容易出现的问题是：

- 音频还是假 URL；
- 发布还是内存态快照；
- 播放装配没有读取真实发布态。

所以改这组接口时，通常不只改 `lesson/service.py`，还要同步看任务状态和发布态对象。

## 八、跨接口公共文件

如果你改的是多类接口共同规则，不要只改某个 domain：

- 统一签名校验：`backend/app/common/security.py`
- 统一异常格式：`backend/app/common/exceptions.py`
- 统一成功响应：`backend/app/common/responses.py`
- 主应用挂载：`backend/app/main.py`

## 九、使用建议

### 当你拿到一个接口任务时

按这个顺序走：

1. 先看 [01-编码入口.md](./01-编码入口.md)
2. 再看 [03-联动顺序与完成定义.md](./03-联动顺序与完成定义.md)
3. 用本文件定位到代码文件
4. 再去 `../api/` 查看该接口契约
5. 行为变化后，必须回写相应文档并持久化

### 行为变化后至少检查这些文档是否要更新

- 对外接口字段变化 -> 更新 `api/` 对应接口文档
- 模块职责变化 -> 更新 `coding-guide/02-后端模块与实现映射.md`
- 联动顺序变化 -> 更新 `coding-guide/03-联动顺序与完成定义.md`
- 当前进度变化 -> 更新 `coding-guide/progress/current-status.md`
