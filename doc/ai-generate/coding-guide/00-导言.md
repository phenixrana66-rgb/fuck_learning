# 代码编写文档层导言

本目录用于承接以下三层文档：

- `../requirements-analysis/`：回答为什么做、做什么、做到什么程度；
- `../project-architecture/`：回答系统如何分层、对象如何流转、模块如何协作；
- `../tech-stack-selection/`：回答使用什么技术、运行时和工程基线。

这层文档不再重复需求背景、架构原理和技术选型理由，而是专门回答一件事：

**基于当前 `backend/` 代码骨架，下一步具体应该怎么写代码、按什么顺序联动、每个模块做到什么程度才算完成。**

## 这层文档适合谁

- 日常写后端代码的人；
- 需要快速进入某个模块继续实现的人；
- 需要减少上下文加载、避免反复翻需求/架构大文档的人。

## 阅读原则

- 平时编码优先从 `01-编码入口.md` 进入；
- 只有遇到边界不清、对象语义不清、技术选型不清时，再回到三层设计文档；
- 本目录默认以当前 `backend/app` 已存在的模块骨架为起点，不重新设计目录结构。

## 当前数据库配置说明（2026-04 更新）

当前 `backend/app/common/config.py` 与 `backend/app/common/db.py` 已经正式消费数据库连接配置，不再只是文档占位。

代码当前支持两种入口：

- 直接使用 `A12_DB_URL`
- 或组合使用 `A12_DB_HOST`、`A12_DB_PORT`、`A12_DB_USER`、`A12_DB_PASSWORD`、`A12_DB_NAME`、`A12_DB_ECHO`

默认运行时会拼出 `mysql+pymysql://...` 连接串；测试与局部验证可通过 `configure_database_url(...)` 切到 SQLite。

注意事项：

- 本文档只保留配置项名称，不再写入真实账号、密码等敏感信息；
- 仓库内与数据库接入相关的代码基线，以 `common/config.py` 与 `common/db.py` 的当前实现为准；
- 当前已落库的范围是 `tasks` 与 `script`，`lesson / qa / progress` 仍未进入数据库持久化。
