# 代码编写文档阅读导航

本目录是 `requirements-analysis/`、`project-architecture/`、`tech-stack-selection/` 之后的“代码编写层”，目标是让日常开发时优先只读取最小必要上下文。

## 文档目录

- [00-导言.md](./00-导言.md) - 本目录定位、边界与使用方式
- [01-编码入口.md](./01-编码入口.md) - 日常编码默认入口，压缩三层设计为最小上下文
- [02-后端模块与实现映射.md](./02-后端模块与实现映射.md) - 当前 `backend/app` 模块与后续实现目标映射
- [03-联动顺序与完成定义.md](./03-联动顺序与完成定义.md) - 模块联动顺序、阶段任务与完成定义
- [05-接口到代码文件映射.md](./05-接口到代码文件映射.md) - 逐接口定位当前 router/schema/service 与后续代码落位
- [06-编码流程与文档持久化规范.md](./06-编码流程与文档持久化规范.md) - 编码开始顺序、文档同步规则与持久化要求
- [07-数据库持久化演进规划.md](./07-数据库持久化演进规划.md) - 从内存态到 PostgreSQL / Redis / MinIO 的落地边界与阶段顺序
- [progress/current-status.md](./progress/current-status.md) - 当前有效进度与默认下一步
- [progress/next-start.md](./progress/next-start.md) - 下次进入项目时的快速启动顺序

## 与 API 文档的关系

- `coding-guide/` 负责：日常编码入口、模块映射、联动顺序、完成定义。
- `api/` 负责：单接口契约、请求/响应字段、链路级接口清单。
- 日常开发时，建议先读 `coding-guide/`，当你开始补某个具体接口时，再跳到 [../api/api-navigation.md](../api/api-navigation.md) 查看对应接口文档。

## 与上游文档关系

- `requirements-analysis/`：负责需求与验收边界
- `project-architecture/`：负责系统结构、对象、模块和状态设计
- `tech-stack-selection/`：负责技术选型、运行时和工程基线
- `coding-guide/`：负责把前三层翻译成日常编码可直接执行的最小上下文
- `api/`：负责把对外接口契约落到逐接口文档，供具体编码和联调时查阅
- `coding-guide/progress/`：负责记录当前有效进度、下次启动顺序与上下文恢复方式
