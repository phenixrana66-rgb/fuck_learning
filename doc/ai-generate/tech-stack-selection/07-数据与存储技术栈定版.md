# 7. 数据与存储技术栈定版
## 7.1 主数据库：MySQL 8.4
MySQL 是本项目的主数据库，统一存储：

- 课件元数据与任务记录；
- `CIR`、节点树、知识锚点；
- 脚本、音频、发布快照；
- 问答、学习进度、续讲计划；
- 部分全文检索索引与检索辅助元信息。

### 选择原因

- 关系建模能力足够支撑当前比赛项目；
- MySQL 部署普及、团队上手成本低；
- `JSON` 字段与 `FULLTEXT` 足以承载首版结构化数据与基础检索；
- 让“关系数据 + 基础检索能力”尽量留在一个主数据库内，减少运维复杂度。

## 7.2 缓存与中间件：Redis
Redis 用途包括：

- Dramatiq broker；
- 热点结构结果缓存；
- 问答结果短缓存；
- 学习会话临时状态；
- 简单限流与去重。

## 7.3 对象存储：MinIO / S3 兼容存储

### 开发环境

`MinIO`：本地开发与演示首选。

### 部署环境

任意 S3 兼容对象存储：用于比赛演示或云环境部署。

### 存储内容

- 原始 PPT / PDF；
- 解析中间产物；
- 脚本导出文件；
- 音频文件；
- 演示截图 / 视频素材（可选）。
