# 10. 部署与环境定版
## 10.1 开发 / 演示部署
正式推荐使用 Docker Compose，最小运行组合如下：

- `nginx`
- `frontend`
- `api`
- `worker`
- `mysql`
- `redis`
- `minio`

## 10.2 服务职责建议

| 服务 | 职责 |
|---|---|
| nginx | 静态资源分发、反向代理、路由收口 |
| frontend | Vue 前端构建产物 |
| api | FastAPI 主服务 |
| worker | 异步任务执行 |
| mysql | 结构化数据与基础全文检索 |
| redis | broker、缓存、会话态 |
| minio | 课件、音频、导出资源 |

## 10.3 为什么不选 Kubernetes
比赛阶段不需要复杂的容器编排治理；

Docker Compose 足以满足演示、联调和基础部署；

运维精力应优先让位于业务闭环与指标达成。
