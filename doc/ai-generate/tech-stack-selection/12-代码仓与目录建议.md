# 12. 代码仓与目录建议
建议采用单仓（monorepo）或统一仓库组织，目录示例如下：

```text
repo/
  frontend/
    src/
      layouts/
      modules/
      api/
      stores/
  backend/
    app/
      platform/
      courseware/
      parser/
      cir/
      script/
      lesson/
      qa/
      progress/
      observability/
      common/
    alembic/
  worker/
    tasks/
  infra/
    docker/
    nginx/
  tests/
    api/
    regression/
    fixtures/
  docs/
```

这样组织的好处是：

- 前后端分层清晰；
- 评测脚本、样本和部署配置可统一管理；
- 后续如果需要拆服务，也能按领域逐步演进。
