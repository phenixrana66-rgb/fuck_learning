# 11. 评测、测试与可观测性定版
## 11.1 测试框架

- `pytest`：接口、服务、回归脚本；
- `JSONL` 黄金集：问答、知识点、续讲样本；
- `Locust`：压测解析轮询和问答接口。

## 11.2 指标与异常监控

- `prometheus-client`：输出 `/metrics` 指标；
- `Sentry`：异常监控与堆栈追踪；
- 结构化 JSON 日志：按 `requestId` 串联链路。

## 11.3 建议重点监控项

- 课件解析耗时；
- 解析成功率；
- 问答响应时延；
- 问答证据命中率；
- TTS 成功率与时长；
- 发布装配成功率；
- 学生端播放恢复成功率。

## 11.4 为什么要把评测栈写进定稿版
因为赛题最终不只看“功能能跑”，还看：

- 时间指标是否达标；
- 答案是否准确；
- 知识点提取是否可靠；
- 是否有可展示的统计表、日志和文档。
