# 6. 异步任务与运行时定版
## 6.1 任务队列：Dramatiq + Redis

### 定稿原因

- 比 Celery 更轻，学习和接入成本更低；
- 对当前项目的异步场景已经足够；
- 更适合比赛阶段的简单部署与稳定调试。

### 进入队列的任务

- 课件解析；
- 结构化后索引构建；
- 检索辅助数据生成；
- 语音合成；
- `LessonPackage` 发布装配；
- 可选的批量回归评测。

## 6.2 同步 / 异步边界

### 同步接口负责

- 查询状态；
- 获取脚本、播放包、会话信息；
- 提问与答复；
- 记录进度；
- 获取续讲计划。

### 异步任务负责

- 解析；
- 索引构建；
- 音频生成；
- 发布装配；
- 大批量内容处理。

## 6.3 为什么不选 Celery / Kafka / Airflow
Celery 虽然成熟，但对当前项目来说心智负担更高；

Kafka / Airflow 更适合更重的分布式与数据编排场景；

当前阶段最需要的是“任务制 + 状态可查 + 易部署”，而不是复杂中间件体系。
