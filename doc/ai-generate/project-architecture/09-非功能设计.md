## 9. 非功能设计
### 9.1 性能与容量

| 指标 | 目标值 | 架构策略 |
|---|---|---|
| 课件解析时间 | <= 2 分钟 / 份 | 异步任务、页级解析、后台索引 |
| 问答响应时间 | <= 5 秒 | 当前节点限域、缓存、混合检索、短上下文 |
| 知识点识别准确率 | >= 80% | 黄金样本集、结构化评测 |
| 答案准确率 | >= 85% | 证据问答、页码回链、人工抽检 |
| 并发人数 | >= 10 | 模块化单体 + Redis + 音频分段缓存 |
| 样本规模 | >= 100 份 | 样本表、编号清单、自动统计 |

### 9.2 安全与合规

- 全链路默认 HTTPS；
- 用户身份、手机号、令牌等敏感字段按最小必要原则收集；
- 课件内容、问答记录、学习记录采用受控存储；
- 教师、学生、平台管理员采用 `RBAC` 权限模型；
- 接口调用日志保留可审计信息；
- 支持脱敏导出演示样本与统计结果。

### 9.3 弱网与终端适配

- 音频按章节拆分，支持分段加载与断点恢复；
- 学生端优先 H5 轻页面，不做重型 3D / 数字人渲染；
- 弱网时允许“文字回答优先、音频延后加载”；
- 播放页必须支持从问答返回当前章节并继续播放。

这里与 `requirements-analysis/10-非功能需求.md` 的一致口径是：当前轮强调轻量化可嵌入和教学闭环稳定，不为了增强表现层牺牲可用性与验收指标。

### 9.4 评测与证据输出

项目最终必须能导出以下材料：

- 解析耗时统计表；
- 问答耗时统计表；
- 知识点识别准确率与答案准确率结果；
- 100 份样本构成表；
- 接口文档与演示链路截图；
- 典型问答案例及来源页码展示。

### 9.5 异常与降级路径

架构层不需要穷举所有业务细节，但必须把会影响接口语义、教学连续性和运行态一致性的异常类型明确下来。

| 场景 | 触发条件 | 架构处理原则 | 对外表现 |
|---|---|---|---|
| 课件解析失败 | 文件不可访问、格式异常、解析超时 | `ParseTask` 进入 `failed`，保留失败原因与请求标识 | 平台可查询失败状态并发起重试 |
| 课件解析部分成功 | 仅部分页或部分结构可识别 | `ParseTask` 进入 `partial_completed`，允许教师先查看可用结构，再决定补跑 | 返回部分结构预览，并提示局部缺失 |
| ASR 低置信度 | 语音识别结果低于可用阈值 | 不直接进入高风险问答；优先提示确认识别文本或改用文字输入 | 问答链路进入 `degraded`，前端可提示重新输入 |
| 证据不足 | 检索不到足够页码/片段支撑回答 | 必须走保守回答，不生成看似确定但不可回链的内容 | 返回保守答案，并明确“当前课件证据不足” |
| 音频或发布装配失败 | 章节音频缺失、装配过程资源不完整 | `LessonPackage` 进入 `publish_failed`，禁止形成伪完整发布态 | 教师侧可见失败状态，学生侧不读取未完成版本 |
| 弱网或音频加载失败 | 带宽不足、音频资源加载超时 | 优先保证文字与结构化内容可继续学习，音频按需延后 | 维持当前章节学习和问答，允许稍后恢复音频 |

### 9.6 异常设计正式约束

- 异常处理优先保证“可查询、可解释、可恢复”，而不是只追求内部吞错。
- 所有降级路径都必须尽量保持教学闭环不断裂，优先保住“继续学、继续问、能续讲”。
- 会影响版本稳定性的失败，不允许直接暴露为学生可学习内容。
- 会影响答案可信度的异常，不允许伪装为正常成功结果。
- 异常与降级策略应服务于评测与答辩举证，因此需要与 `requestId`、任务状态和典型案例留痕联动。
